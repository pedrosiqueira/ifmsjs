Operadores lógicos

Um bug muitas vezes entra por uma porta que você não sabia que deixou aberta.

Considere o seguinte trecho de código para verificar se uma idade está entre 18 e 64 anos:

if (idade >= 18) {
    if (idade < 65) {
        // Your code goes here
    }
}

Embora funcione, instruções aninhadas não são muito legíveis. Utilizando operadores lógicos, podemos simplificar esse tipo de verificação.

Exemplos de operações lógicas:

  • SE estiver com sede E tiver água, ENTÃO beba!
  • SE tiver carro OU tiver carona, ENTÃO vá ao show!
  • SE NÃO tiver máscara, ENTÃO fique em casa!
  • SE tiver nota boa E NÃO faltou, ENTÃO aprovou!
  • SE NÃO tiver guarda-chuva E chover, ENTÃO espere a chuva passar!

Operador Conjuntivo

Para assegurar que duas condições são verdadeiras, utilize o operador && (E condicional):

if (idade >= 21 && idade < 65) { ... }

Apesar de opcional, parênteses redundantes podem tornar a expressão mais clara:

if ((idade >= 21) && (idade < 65)) { ... }

Operador Disjuntivo

Para assegurar que uma das duas ou ambas as condições são verdadeiras, utilize o operador || (OU condicional).

if (horas > 40 || vendas > 25000.00) { ... }

Operador de negação

O operador unário ! (NÃO lógico) “inverte” o valor de uma expressão lógica.

O operador de negação é colocado antes de uma condição.

if (!masculino) { ... }

Ao negar uma expressão com outros operadores, ela deve estar envolta entre parênteses, pois o operador de negação tem maior prioridade. Portanto, as duas expressões seguintes não são equivalentes:

if (! (n % 2 == 0)) { ... }
if (! n % 2 == 0) { ... }

Na maioria dos casos, é possível evitar o operador de negação mudando o operador relacional e tornando a expresão mais legível:

if (n % 2 != 0) { ... }

Qual seria a versão do seguinte trecho de código sem usar o operador !?

if (! n <= 0) { ... }

Os operadores lógicos geram o valor true para verdadeiro e o valor false para falso.

if (idade >= 21 && idade < 65) { ... }
if (horas > 40 || vendas > 25000.00) { ... }
if (!masculino) { ... }

Uma condição que usa os operadores && ou || utiliza a avaliação em curto-circuito — a avaliação termina assim que encontra a resposta.

Considere a expressão

(i != 0) && (10 / i == 2)

A condição dependente (10 / i == 2) deve aparecer após o operador && para evitar a possibilidade de divisão por zero.

Suponha que a escola queira recompensar o alunos que tenham bom desempenho e tenham perdido poucas aulas. Além disso, a recompensa exige que os alunos ingressem em três projetos escolares ou participem de duas atividades esportivas. A avaliação poderia ficar assim:

if (nota > 9 && faltas <= 3 && (projetos >= 3 || esportes >=  2)) {
    ...
}

Para tornar o código mais legível, você pode acrescentar quebras de linhas entre os operadores:

if (nota > 9 && faltas <= 3
&& (projetos >= 3 || esportes >= 2)) {
    ...
}
