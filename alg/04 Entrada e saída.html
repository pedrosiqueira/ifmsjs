Entrada e saída

Quem não comunica se trumbica. – Chacrinha.

Comunicação com o mundo exterior

Até agora, aprendemos algumas instruções do Javascript para realizar operações aritméticas. No entanto, nossos programas são como caixas fechadas - tudo o que acontece dentro deles permanece lá, sem interação com o mundo exterior. Para tornar nossos programas úteis, precisamos de uma maneira de fazê-los enviar e receber informações do mundo exterior. Isso pode incluir outros programas, arquivos que o programa lê ou escreve e, é claro, o usuário - a pessoa interagindo com o programa que desenvolvemos.

Portanto, um programa pode ser visto como uma caixa preta que recebe dados de entrada, os processa e produz uma saída. Para quem interage com o programa, não é necessário saber como ele realiza o processamento - basta fornecer os dados de entrada e esperar pela saída correta. Podemos comparar esse processo com a respiração: inspiramos os gases da atmosfera, nosso corpo os processa e exala os gases alterados.

Outra analogia comum é a culinária. Para fazer um bolo, seguimos uma receita (algoritmo) para combinar os ingredientes (entrada) e produzir um bolo (saída). Os ingredientes e o bolo final não fazem parte da receita em si - eles vêm de fora.

Saída de dados

O comando console.log é usado e para exibir informações no terminal. Dizemos "escrever no terminal" ou simplesmente escrever. Para usar o comando console.log, basta escrever console.log() e colocar o conteúdo que deseja exibir entre os parênteses. Por exemplo:

console.log("Olá, mundo!")

Isso exibirá a mensagem "Olá, mundo!" no terminal.

Você também pode usar o comando console.log para exibir o valor de variáveis. Por exemplo:

let x = 10
console.log(x)

Isso exibirá o valor da variável x, que é 10, no terminal.

Como uma expressão aritmética gera um resultado, você pode colocá-la dentro do console.log para exibir diretamente o resultado da expressão:

let x = 10
console.log(x * 3 / 2 - x) // 5.0

Você pode exibir vários dados em um único comando console.log, separando-os por vírgulas. Neste caso, o comando automaticamente insere um espaço entre os dados, exibindo tudo em uma única linha. Por exemplo:

let x = 5
let y = 10
console.log("A soma de", x, "e", y, "é", x + y)

Isso exibirá A soma de 5 e 10 é 15. Observe que o comando console.log inseriu espaços entre cada argumento para criar uma mensagem bem formatada.

O comando console.log é de uma categoria de comandos chamada de função. Pense em uma função como se fosse uma expressão aritmética, onde os operandos são os argumentos e os operadores são as vírgulas. Os argumentos de uma função sempre devem estar entre parênteses, precedidos pelo nome da função. E, semelhante a uma expressão aritmética, a maioria das funções também resulta em um valor, que pode ser atribuído a uma variável. Chamamos esse resultado de valor retornado ou simplesmente retorno. Entretanto, a função console.log é uma exceção, já que não resulta em um valor que possa ser atribuído a uma variável. Tomei a liberdade de adiantar um pouquinho o assunto de funções, mas, por hora, você não precisa entender a fundo esse conceito, o estudaremos mais para frente. Basta saber que certos comandos que usam parênteses são chamados de funções, que argumentos separados por vírgulas vão dentro dos parênteses das funções, e que as funções podem retornar valores.

A função console.log é uma ferramenta útil para exibir informações durante a execução de um programa e ajudar a entender como ele está funcionando.

Entrada de dados

O comando scanf do pacote node-scanf pode ser usado para receber informações do teclado facilmente. Dizemos "ler do teclado" ou simplesmente ler. Ele é baseado no comando homônimo da linguagem de programação C. A sintaxe do comando é a seguinte:

variavel = scanf("tipo")

Onde variavel é o nome da variável que receberá o valor a ser lido do teclado, e tipo é um argumento que representa o tipo do valor a ser lido. scanf suporta a leitura dos seguintes tipos de dados:

  • %d - int
  • %f - float
  • %s - string
  • %S - string of line
  • %x - hex
  • %a - hex float
  • %o - octal

Por exemplo:

import scanf from "scanf";

let x = scanf("%s")

(Observação: ao usar qualquer recurso de um pacote de terceiro, como o comando scanf, é necessário, antes, importá-lo ao nosso código, como feito na linha 1. Por brevidade, vamos omitir o comando import, mas ele sempre se faz necessário ao usar o scanf).

Este comando "congela" o programa, aguardando que o usuário digite algum valor e pressione a tecla Enter. Após isso, o que o usuário digitou será atribuído (armazenado) à variável x.

O comando scanf também é uma função, mas diferentemente do console.log, resulta em um valor, que é aquilo que o usuário digitou. Observe que o valor lido pelo scanf deve ser compatível com o que ele está esperando, senão, a leitura não é realizada corretamente:

console.log("Please input a number");
let number = scanf("%d");
console.log("your number is", number);

Também tome cuidado para não confundir os tipos dos valores sendo lidos:

console.log("Enter two numbers:")
let n1 = scanf("%s")
let n2 = scanf("%s")
let soma = n1 + n2
console.log("The sum of the numbers is", soma)

O operador de adição, quando operado com pelo menos uma string, realiza a operação de concatenação, que é "engatar" o final do primeiro operando no início do segundo operando.

Portanto, se você quiser que o JavaScript interprete o retorno do scanf como um número inteiro, deve usar o argumento %d. Experimente trocar o %s por %d no código anterior.

Da mesma forma, use o argumento %f para ler um número float:

console.log("What's your height?")
let h = scanf("%f")
console.log("great, you measure", h)

Juntos, os comandos scanf e console.log podem ser usados para criar simples programas interativos que se comunicam com o usuário.

Concatenação

Você pode concatenar qualquer tipo de dados a uma string usando o operador +. O Javascript tentará converter o tipo de dados em uma string para fazer a concatenação. No seguinte exemplo, os tipos primitivos foram concatenados à uma string:

console.log('palavra' + 123 + 65.4 + false)

Cuidado! Como o + é avaliado da esquerda para a direita, o segundo + realiza a concatenação, e não a soma:

const y = 5
console.log("y + 2 = " + y + 2)

Se quiser que o segundo operador realize a soma, deve colocá-lo entre parênteses:

const y = 5
console.log("y + 2 = " + (y + 2))

Lendo múltiplos valores de uma vez

Você também pode ler vários valores em um único scanf:

import scanf from 'scanf';

console.log('digite três inteiros')
let inteiros = scanf('%d%d%d')
console.log('vc digitou', inteiros);

Você pode digitar três inteiros separados por espaço ou enter. Mas os três valores ficam numa única variável, que é uma lista. Mais para frente estudaremos listas na programação. Mas, já adiantando um pouco, para você desagrupar cada valor em uma única variável, você pode usar o recurso de destruturação de listas:

import scanf from 'scanf';

console.log('digite três inteiros')
let [n1, n2, n3] = scanf('%d%d%d')
console.log('vc digitou', n1, n2, n3)
let soma = n1 + n2 + n3
console.log('a soma dos números é', soma);

Os números lidos devem ser separados por espaço ou enter (quebra de linha), por padrão. Mas você pode alterar esse comportamento informando uma "máscara" ao scanf. Por exemplo, uma data é composta por três inteiros, representando, respectivamente, o dia, mês e ano. Para o usuário digitar uma data no formato dd/mm/aaaa, o código pode ficar assim:

import scanf from 'scanf';

console.log('digite uma data no formato dd/mm/aaaa')
let [d, m, a] = scanf('%d/%d/%d')
console.log('a data informada é', d, '/', m, '/', a)

Mais um pouco sobre comentários

Computadores interpretam e executam códigos, mas eles não têm a capacidade de adivinhar tua intenção. Um simples erro no código, como não fechar um parêntese, e teu computador não saberá o que fazer.

Além disso, a programação muitas vezes é uma atividade colaborativa. Em uma empresa, o programa é desenvolvido em equipe. É essencial garantir que teu código seja correto tanto para a máquina quanto compreensível para outros programadores.

À arte de explicar o que teu código faz, damos o nome de documentação. Uma maneira comum de documentar sistemas é adicionar comentários diretamente no código-fonte…

Todas as linguagens de programação permitem que você insira comentários em teu código. Comentários são linhas que fazem parte do código, mas são ignoradas pelo computador. Eles servem exclusivamente para os programadores explicarem o propósito de determinadas partes do código.

Dica: tenha o hábito de adicionar comentários à medida que escreve o código.

Tipos de comentários

Há dois tipos de comentários: comentários simples e comentários em bloco.

Os comentários simples começam com duas barras (//) e se estendem até o final da linha. Geralmente, são usados para pequenas anotações que cabem em uma única linha.

Os comentários em bloco começam com barra e asterisco (/*) e terminam com asterisco e barra (*/). Eles são usados para escrever comentários mais extensos que ocupam várias linhas.

/*
Programa de exemplo de uso de comentários.
Autor: Pedro Henrique
Data: 08/08/2018
*/

// Solicita ao usuário que digite dois números
console.log("Digite dois números...");

// Lê os números que o usuário digita
const [a, b] = scanf("%d%d")

// Determina qual número é maior e qual é menor
const maior = a > b ? a : b;
const menor = a > b ? b : a;

// Escreve o maior e o menor número
console.log("Maior:", maior);
console.log("Menor:", menor);

Você pode não entender todos os comandos deste programa, mas ao ler os comentários, você pode ter uma ideia do que cada um deles está fazendo.

Principalmente em nossos primeiros programas, é comum incluir muitos comentários para ajudá-lo a se familiarizar com JavaScript.

Entretanto, tenha cuidado para não sobrecarregar teu código com comentários redundantes ou desnecessários, como o exemplo a seguir:

// Exibe "Gosto de programar"
console.log("Gosto de programar")

Dica: Comentários podem ser usados para testar teu código ou ignorar “temporariamente” um erro no teu código! Colocar duas barras (//) no início de uma instrução fará com que o computador a ignore!

Exercícios

Para cada exercício, elabore um programa que resolva à questão.

  1. Dadas a base $b$ e a altura $a$ de um retângulo, qual sua área? $A=b\cdot a$
  2. Dado o raio de um círculo, qual sua área? $A=πr^2$
  3. Dado um inteiro, qual o seu antecessor e o seu sucessor?
  4. Dados dois inteiros, quantos inteiros há no intervalo fechado entre eles?
  5. A família Silva vai passar as férias na Europa e quer saber quantos euros vale o dinheiro em real que ela levará. Dados um valor na moeda real e a cotação do euro, qual o valor correspondente em euros?
  6. Joãozinho fez sua viagem até os Estados Unidos. Porém, chegando lá, descobriu que a temperatura é medida em graus Fahrenheit, e não em Celsius. Dada a temperatura em graus Fahrenheit, qual a temperatura correspondente em graus Celsius? $C=\dfrac{(F-32)\cdot 5}{9}$
  7. Dadas quatro notas de um estudante, qual sua média aritmética?
  8. Dados o salário de um funcionário e o percentual de aumento, qual o valor do aumento e o novo salário?
  9. Ler dois números e armazená-los nas variáveis $a$ e $b$. Em seguida, trocar o valor das variáveis, ou seja, $a$ deve receber o valor de $b$ e vice-versa. Por fim, escrever os valores de $a$ e $b$. (obs.: essa é a operação de troca, ou swap, e é muito utilizada em vários algoritmos)
  10. Joãozinho fará uma viagem de carro e quer saber quantos $\$$ ele gastará em gasolina. Dados três decimais $q$, $d$, $p$, que são, respectivamente, o consumo do carro medido em quilômetros por litro (Km/l), a distância total que será percorrida (Km) e o preço do litro da gasolina ($\$$/l), quantos $\$$ Joãozinho gastará em gasolina para viajar?
