Estruturas de seleção

Em geral, as instruções em um programa são executadas sequencialmente, uma após a outra, seguindo a ordem em que são escritas. Esse fluxo é conhecido como estrutura sequencial.

No entanto, existem situações em que é necessário especificar que a próxima instrução a ser executada não seja necessariamente a próxima na sequência. Para lidar com essas situações, utilizamos a estrutura de seleção.

As instruções if e else tornam seus programas mais inteligentes, permitindo a eles tomar decisões baseadas em certas condições.

Você também aprenderá a usar os operadores relacionais, que servem para comparar valores, e são usados em conjunto com a instrução if.

As instruções if e else funcionam como na linguagem falada: se algo é verdadeiro, faça uma coisa; caso contrário, faça outra coisa.

  • SE eu ganhar na loteria, ENTÃO irei para a Itália; SENÃO, continuarei em 3L.
  • SE chover, ENTÃO leve um guarda-chuva.
  • SE tua nota for menor que 7, ENTÃO você ficou de recuperação; SENÃO, você foi aprovado.

Operadores relacionais

Os operadores relacionais comparam dois valores. Você sempre coloca uma variável, literal ou expressão em ambos os lados de um operador relacional.

Operador Descrição Exemplo
> Maior que numero > 10
< Menor que 10 * x < 8 - y
>= Maior ou igual a 9 >= 19.7
<= Menor ou igual a c <= 'e'
== Igual a i == j
!= Diferente de 'z' != 'g'

Um operador sempre resulta em um valor. Um operador relacional resulta no valor true quando o resultado da comparação é verdadeiro e resulta no valor false quando o resultado da comparação é falso.

ATENÇÃO:

  • Não confunda = com ==, lembre-se que um sinal de igual é atribuição e dois sinais de igual é comparação.
  • É recomendado comparar somente valores equivalentes (mesmo tipo), para evitar comportamentos inesperados. (não compare letra com número, por exemplo...)
  • Ao comparar booleanos com números, o valor booleano true é equivalente ao número 1 e o valor booleano false é equivalente ao número 0.
  • Não inverta os caracteres dos operadores (=>, =<,=!, ..., é errado!).
  • NÃO FAÇA OPERAÇÕES RELACIONAIS COMPARANDO MAIS DE DOIS VALORES: X < Y < Z não funciona como na matemática.

Seleção simples

if (condição) {
    sequência de instruções
    que compõem o bloco do if
}

A condição é uma expressão cujo resultado é um booleano ou é interpretado como tal. Se não for booleano, os seguintes valores são interpretados como false:

  • undefined, null, 0, -0, 0n, NaN, "".

Qualquer outro valor é considerado como true.

Um bloco de código é uma sequência de instruções dentro de chaves ({ e }). Também chamamos o bloco de corpo.

Um recuo, ou indentação, se refere a uma certa quantidade de espaços ou tabulação que se coloca no início de cada instrução de um bloco de código. Por convenção, no Javascript, se usa recuos de 4 espaços.

Caso a condição seja true, o corpo do if é executado. Caso contrário, o corpo do if não é executado, e o programa continua da próxima instrução depois do corpo do if.

Caso um bloco de código for composto por apenas uma linha, as chaves são opcionais.

Execute esse código e teste tanto com valores que atendam às condições do if quanto que não atendam. Procure entender o que está acontecendo no código:

import scanf from "scanf";

console.log("\nInforme teu salario: ");
let salario = scanf("%f")

console.log("\nVocê ganha", salario);
if (salario > 5000) {
    console.log("Você está bem, ein!");
}

console.log("\nInforme tua nota: ");
let nota = scanf("%f")

console.log("\nVocê tirou nota", nota);
if (nota < 7)
{
    console.log("Você está de recuperação!");
}

console.log("\nInforme teu numero da sorte: ");
let numero = scanf("%d")

console.log("\nTeu número da sorte é", numero);
if (numero % 2 == 0) console.log("Número par");

if (numero % 5 == 0) 
    console.log("Número múltiplo de 5.");

console.log("\nInforme tua idade: ");
let idade = scanf("%d")

console.log("\nTua idade é", idade);
if (idade >= 18) {
    console.log("Você é maior de idade,");
    console.log("Já é responsável legalmente.");
}

console.log("\nFim do programa.");

Seleção composta

A instrução else é o irmão que complementa o if.

Todo else deve vir acompanhado de um if.

O corpo do if é executado caso a condição seja verdadeira. Caso a condição do if seja falsa, aí é o corpo do else que é executado.

Execute esse código e teste com valores que explorem tanto o if quanto o else. Procure entender o que está acontecendo no código:

import scanf from "scanf";

console.log("\nInforme teu salario: ");
let salario = scanf("%f")

console.log("\nVocê ganha", salario);
if (salario > 5000) {
    console.log("Você está bem, ein!");
} else {
    console.log("Você ainda chega lá!");
}

console.log("\nInforme tua nota: ");
let nota = scanf("%f")

console.log("\nVocê tirou nota", nota);
if (nota < 7)
{
    console.log("Você está de recuperação!");
}
else
    console.log("Você passou de ano!");

console.log("\nInforme teu numero da sorte: ");
let numero = scanf("%d")

console.log("\nTeu número da sorte é", numero);
if (numero % 2 == 0) console.log("Número par");
else console.log("Número ímpar");

if (numero % 5 == 0) 
    console.log("Número múltiplo de 5.");
else
    console.log("Número NÃO múltiplo de 5.");

console.log("\nInforme tua idade: ");
let idade = scanf("%d")

console.log("\nTua idade é", idade);
if (idade >= 18) {
    console.log("Você é maior de idade,");
    console.log("Já é responsável legalmente.");
}
else
{
    console.log("Você ainda é menor de idade");
}

console.log("\nFim do programa.");

Apesar de eu ter feito variações de blocos na mesma linha e na linha de baixo, a convenção é sempre iniciar o bloco na linha de baixo, mesmo que seja composto por apenas uma linha.

Instruções aninhadas

Você pode colocar ifs e elses dentro de ifs e elses.

Quando colocamos instruções dentro de instruções, chamamos de instruções aninhadas.

Considere o seguinte exercício: ler três números e escrever se o maior número é par ou ímpar.

Solução 1

import scanf from "scanf";

console.log("Digite 3 números...");
let x = scanf("%d");
let y = scanf("%d");
let z = scanf("%d");

if (x > y) {
    if (x > z) {
        console.log(x, "é o maior número");
        if (x % 2 != 0) {
            console.log("e também é ímpar");
        } else {
            console.log("e também é par");
        }
    } else {
        console.log(z, "é o maior número");
        if (z % 2 != 0) {
            console.log("e também é ímpar");
        } else {
            console.log("e também é par");
        }
    }
} else {
    if (y > z) {
        console.log(y, "é o maior número");
        if (y % 2 != 0) {
            console.log("e também é ímpar");
        } else {
            console.log("e também é par");
        }
    } else {
        console.log(z, "é o maior número");
        if (z % 2 != 0) {
            console.log("e também é ímpar");
        } else {
            console.log("e também é par");
        }
    }
}

Observe que o if da linha 11 está dentro do if da linha 9 que por sua vez está dentro do if da linha 8. Isso quer dizer que para a linha 12 ser executada, os ifs das linhas 8, 9 e 11 devem ser verdadeiros.

Há várias formas de fazer um programa. Nesta, optei por usar ifs e elses. Daria para fazer apenas com ifs, porém ficaria mais complexo.

Solução 2

import scanf from "scanf";

console.log("Digite 3 números...");
let x = scanf("%d");
let y = scanf("%d");
let z = scanf("%d");
let maior;

if (x > y) {
    if (x > z) {
        maior = x;
    } else {
        maior = z;
    }
} else {
    if (y > z) {
        maior = y;
    } else {
        maior = z;
    }
}

console.log(maior, "é o maior número");

if (maior % 2 != 0) {
    console.log("e também é ímpar");
} else {
    console.log("e também é par");
}

Aqui está o mesmo programa feito de forma diferente. Para economizar linhas, armazenei o maior número numa variável e no final verifiquei se essa variável é par ou ímpar, ao invés de fazer a verificação várias vezes.

Solução 3

import scanf from "scanf";

console.log("Digite 3 números...");
let x = scanf('%d');
let y = scanf('%d');
let z = scanf('%d');
let maior;

if (x > y) {
    maior = x;
} else {
    maior = y;
}

if (z > maior) {
    maior = z;
}

console.log(maior, "é o maior número");

if (maior % 2 != 0) {
    console.log("e também é ímpar");
} else {
    console.log("e também é par");
}

O mesmo exercício resolvido de mais outra forma. Dessa vez, usando ainda menos if-else’s.

O problema do else oscilante

O Javascript sempre associa um else à instrução if imediatamente anterior, a menos que instruído de outro modo pela colocação de chaves ({ e }). Esse comportamento pode levar àquilo que é chamado de problema do else oscilante.

Cuidado! Isso daqui:

if (x > 5)
    if (y > 5)
        console.log("x e y são maiores que 5");
else
    console.log("x é menor que 5");

É equivalente a isso daqui:

if (x > 5)
    if (y > 5)
        console.log("x e y são maiores que 5");
    else
        console.log("x é menor que 5");

Use chaves para associar o else ao primeiro if deste exemplo…

Instrução vazia

Uma instrução vazia é representada colocando um ponto-e-vírgula (;) onde uma instrução normalmente estaria. Colocar um ponto-e-vírgula depois da condição em uma instrução if resulta em um erro de lógica em um if único ou em erro de sintaxe em um if..else. Por exemplo,

if (number1 == number2); // erro de lógica
    console.log(number1, "==", number2);

É equivalente a:

if (number1 == number2)
    ; // estrutura vazia

console.log(number1, "==", number2);

Erros

Um erro de sintaxe ocorre quando o código que você escreve não segue a gramática definida pela linguagem de programação. Esses erros são detectados durante a análise do código, antes de ser executado. Eles geralmente resultam em mensagens de erro que indicam a linha e o local onde a violação ocorreu. Erros de sintaxe incluem coisas como escrever palavras-chave erradas, omitir parênteses de fechamento, usar operadores de forma incorreta e outros problemas que violam as regras da linguagem.

Quantos erros de sintaxe você consegue encontrar a seguir?

console.log("tudo certo??")

console.log("qual tua nota? )
nota1 = scanf(%f)
if nota2 >= 6 {
    print("aprovado"
else 
    print(reprovado)
}

Erros de sintaxe também podem ser chamados de erros em tempo de compilação ou simplesmente erros de compilação, porque eles são detectados pelo compilador antes da execução do programa, impedindo o programa de ser executado, mesmo que apenas algumas instruções estejam erradas.

Um erro em tempo de execução ou simplesmente erro de execução ocorre quando o programa está em execução e encontra uma situação imprevista que impede seu funcionamento normal. Isso pode ser causado por vários motivos, como entradas inválidas, tentativas de acesso a variáveis inexistentes, divisões por zero, entre outros. Esses erros são frequentemente chamados de exceções e interrompem a execução do programa no momento em que ocorrem, geralmente exibindo uma mensagem de erro para o usuário.

Nesse exemplo, se o usuário digitar erro, ocorrerá um erro de execução:

import scanf from "scanf";

let string = scanf("%s")

if (string == 'erro')
    string = 3
console.log(string.toUpperCase())

console.log("Deu tudo certo!")

Um erro de lógica ocorre quando um programa produz resultados indesejados devido a problemas em sua lógica interna ou na implementação do algoritmo. Esse tipo de erro não é detectado pelo interpretador ou compilador como um erro, porque o código está sintaticamente correto e não resulta em falhas de compilação ou execução. No entanto, o programa pode não realizar as operações desejadas ou pode produzir resultados incorretos.

Considere o seguinte programa que não calcula corretamente o quadrado de um número, resultando em um erro de lógica:

console.log("Programa para calcular o quadrado de um número...")
console.log("Digite um número: ")
numero = scanf("%f")
quadrado = numero * 2
console.log("O quadrado do número é", quadrado)

Seleção encadeada

Considere o seguinte exercício:

A nota final de um estudante é calculada a partir de três notas atribuídas, respectivamente, a um trabalho, a um seminário e a uma prova. A média das três notas mencionadas obedece aos pesos da tabela A.

Dadas as três notas de um estudante, qual sua média ponderada e seu conceito conforme a tabela B?

Tabela A
Nota Peso
Trabalho 2
Seminário 3
Prova 5
Tabela B
MÉDIA PONDERADA CONCEITO
Maior ou igual a 8,5 A, aprovado :D
Maior ou igual a 7 e menor que 8,5 B, aprovado :)
Maior ou igual a 5 e menor que 7 C, recuperação :|
Maior ou igual a 2,5 e menor que 5 D, recuperação :(
Menor que 2,5 E, reprovado :O

Solução 1:

import scanf from "scanf";

console.log("Boletim Digital");
console.log("###############\n");

console.log("Trabalho:");
let t = scanf("%f");
console.log("Seminário:");
let s = scanf("%f");
console.log("Prova:");
let p = scanf("%f");

let m = (t * 2 + s * 3 + p * 5) / 10;

console.log("\nMédia:", m);

if (m >= 8.5) {
    console.log("Conceito A, aprovado :D");
} else {
    if (m >= 7) {
        console.log("Conceito B, aprovado :)");
    } else {
        if (m >= 5) {
            console.log("Conceito C, recuperação :|");
        } else {
            if (m >= 2.5) {
                console.log("Conceito D, recuperação :(");
            } else {
                console.log("Conceito E, reprovado :O");
            }
        }
    }
}

Embora esse código funcione, esse aninhamento de ifs e elses torna o código menos legível. Apesar da recomendação ser sempre colocar chaves, em casos como esse, o código fica mais legível ao omitir as chaves.

Solução 2:

import scanf from "scanf";

console.log("Boletim Digital");
console.log("###############\n");

console.log("Trabalho:");
let t = scanf("%f");
console.log("Seminário:");
let s = scanf("%f");
console.log("Prova:");
let p = scanf("%f");

let m = (t * 2 + s * 3 + p * 5) / 10;

console.log("\nMédia:", m);

if (m >= 8.5) {
    console.log("Conceito A, aprovado :D");
} else if (m >= 7) {
    console.log("Conceito B, aprovado :)");
} else if (m >= 5) {
    console.log("Conceito C, recuperação :|");
} else if (m >= 2.5) {
    console.log("Conceito D, recuperação :(");
} else {
    console.log("Conceito E, reprovado :O");
}

As instruções encadeadas if e else são mutuamente exclusivas. Isto é, apenas o corpo da primeira instrução verdadeira é executado. Se nenhuma for verdadeira, e houver um else, apenas o corpo do else é executado.

Operador ternário

Em Javascript, assim como em algumas outras linguagens de programação, você pode usar o operador ternário para expressar uma operação condicional de forma mais concisa. O operador ternário é uma forma compacta de escrever um if-else de uma linha.

A sintaxe básica do operador ternário em Javascript é a seguinte:

condição ? valor_se_verdadeiro : valor_se_falso

A condição é a expressão que será avaliada. Se for verdadeira, o resultado da expressão é valor_se_verdadeiro; caso contrário, o resultado da expressão é valor_se_falso.

Aqui está um exemplo de código que utiliza operador ternário para determinar o maior entre dois números:

let a = scanf("%d")
let b = scanf("%d")

maior = a > b ? a : b;
console.log(maior)

Neste caso, o operador ternário verifica se a é maior que b. Se for verdadeiro, a variável maior recebe o valor de a, senão, recebe o valor de b. Isso é equivalente à:

let a = scanf("%d")
let b = scanf("%d")

if (a > b)
    maior = a
else
    maior = b

console.log(maior)

Considere também este exemplo:

console.log("Qual é a tua nota?")
let nota = scanf("%f")
console.log(nota >= 6 ? "Aprovado!" : "Reprovado!")

O operador ternário é útil para expressões condicionais simples, mas deve ser usado com moderação para não comprometer a legibilidade do código.
