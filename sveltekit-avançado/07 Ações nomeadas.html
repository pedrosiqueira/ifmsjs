Ações Nomeadas

Em SvelteKit, além de uma única ação padrão por página, podemos definir múltiplas ações nomeadas. Isso nos permite gerenciar diferentes submissões de formulário na mesma rota, de forma organizada e centralizada no arquivo +page.server.js. Podemos declarar várias funções como propriedades de um objeto actions. Cada função é invocada ao especificarmos seu nome na URL, como ?/nomeDaAcao, no atributo action de um formulário ou formaction de um botão.

Vamos ilustrar com um exemplo de uma calculadora que pode multiplicar e dividir. Crie o arquivo src/routes/07/named/+page.server.js:

import { fail } from '@sveltejs/kit';

// O objeto `actions` exportado contém todas as ações nomeadas da página.
// Cada chave do objeto (ex: 'multiplicar') é o nome da ação.
export const actions = {
  // Ação 'multiplicar'
  multiplicar: async ({ request }) => {
    const data = await request.formData();
    const x = Number(data.get('x'));
    const y = Number(data.get('y'));

    // Validação: garante que os campos são números.
    if (isNaN(x) || isNaN(y)) {
      // Retorna um erro 400 (Bad Request) com uma mensagem e os dados submetidos.
      return fail(400, { error: 'Informe números válidos.', x, y });
    }

    // Sucesso: retorna um objeto com o resultado.
    return { result: x * y };
  },

  // Ação 'dividir'
  dividir: async ({ request }) => {
    const data = await request.formData();
    const x = Number(data.get('x'));
    const y = Number(data.get('y'));

    // Validação 1: garante que os campos são números.
    if (isNaN(x) || isNaN(y)) {
      return fail(400, { error: 'Informe números válidos.', x, y });
    }
    
    // Validação 2: impede a divisão por zero.
    if (y === 0) {
      return fail(400, { error: 'Não é possível dividir por zero!', x, y });
    }

    // Sucesso: retorna um objeto com o resultado.
    return { result: x / y };
  }
};

Explicação:

  • Declaramos um objeto actions que contém duas propriedades: multiplicar e dividir. Cada propriedade corresponde a uma função assíncrona que processa os dados do formulário.
  • Mesmo que os dados do formulário representem números, eles chegam como strings. Por isso, usamos a função Number para converter essas strings em valores numéricos.
  • Realizamos validações para garantir que os dados convertidos sejam, de fato, números. Caso o usuário tenha digitado algo inválido, a função Number não conseguirá fazer a conversão, e a função isNaN (de is not a number) retornará true, indicando a entrada inválida.
  • Utilizamos a função fail(status, data) do SvelteKit para retornar uma resposta de erro controlada. Isso evita que o aplicativo quebre e permite exibir uma mensagem amigável ao usuário, preservando os dados que ele já preencheu.

Agora, crie o arquivo src/routes/07/named/+page.svelte para a interface:

<script>
  let { form } = $props();
</script>

<form method="POST" action="?/multiplicar">
  <label>
    x: <input name="x" type="number" step="0.01" value={form?.x ?? ''} required />
  </label>
  <br />
  <label>
    y: <input name="y" type="number" step="0.01" value={form?.y ?? ''} required />
  </label>
  <br />
  <button type="submit">Multiplicar</button>
  <button type="submit" formaction="?/dividir">Dividir</button>
</form>

{#if form && form.error != undefined}
  <p style="color: red">{form.error}</p>
{/if}

{#if form && form.result != undefined}
  <p style="color: green">Resultado: {form.result}</p>
{/if}

Explicação:

  • Um único formulário pode executar lógicas completamente diferentes dependendo de qual botão é clicado.
  • O formulário principal tem action="?/multiplicar". Isso significa que, por padrão (por exemplo, se o usuário pressionar Enter no campo de input), a ação multiplicar será executada.
  • O botão "Dividir" possui o atributo formaction="?/dividir". Ele sobrepõe o action do formulário, direcionando a submissão para a ação dividir.
  • Os valores de form.x e form.y são preservados em caso de erro para facilitar a correção.
  • Se form.error existir, exibimos a mensagem de erro; se form.result existir, exibimos o resultado.

O atributo action (ou formaction) envia o formulário para a URL atual, apenas adicionando o parâmetro de busca (ex: ?/multiplicar). Se precisar enviar os dados para uma rota completamente diferente, você pode especificar o caminho completo: action="/outra/rota?/nomeDaAcao".

Atenção: Não é recomendável combinar ação padrão com ações nomeadas na mesma página. Se você submeter uma ação nomeada sem redirecionar, o parâmetro ?/acao permanecerá na URL. Assim, uma próxima submissão para a ação padrão continuará sendo tratada pela última ação nomeada, o que pode causar comportamentos inesperados.

Exercícios Práticos

  1. Você está desenvolvendo a área "Minha Conta" de um aplicativo web. Nesta página, o usuário logado deve ser capaz de realizar três ações distintas em um mesmo formulário: atualizar suas informações de perfil, alterar sua senha e desativar sua conta.

    Crie o arquivo src/routes/07/perfil/+page.svelte com um formulário com os seguintes campos e botões:

    • Um campo para Nome
    • Um campo para E-mail
    • Um campo para Senha Atual
    • Um campo para Nova Senha
    • Um campo para Confirmar Nova Senha
    • Uma caixa de seleção para confirmar a desativação da conta
    • Botão "Salvar Alterações
    • Botão "Alterar Senha"
    • Botão "Desativar Minha Conta

    Crie o arquivo src/routes/07/perfil/+page.server.js com as seguintes ações nomeadas:

    1. Ação atualizarPerfil:

      • Acionada por: Botão "Salvar Alterações".
      • Validação: Os campos nome, e-mail e senha não podem estar vazios. A senha deve ser 'senhasecreta'.
      • Em caso de falha: Use fail(400, { ... }) para retornar uma mensagem de erro e os dados preenchidos nos campos de nome e e-mail.
      • Em caso de sucesso: Use redirect(303, '/07/perfil?status=perfil_atualizado') para recarregar a página e mostrar uma mensagem de sucesso.

    2. Ação alterarSenha:

      • Acionada por: Botão "Alterar Senha".
      • Validação:
        • Os três campos de senha não podem estar vazios.
        • A "Nova Senha" e a "Confirmar Nova Senha" devem ser idênticas.
        • A "Nova Senha" deve ter no mínimo 8 caracteres.
        • A senha atual deve ser 'senhasecreta'.
      • Em caso de falha: Use fail(400, { ... }) com uma mensagem de erro específica para cada problema de validação. Não retorne as senhas preenchidas por segurança.
      • Em caso de sucesso: Use redirect(303, '/07/perfil?status=senha_alterada').

    3. Ação desativarConta:

      • Acionada por: Botão "Desativar Minha Conta".
      • Validação: A caixa de seleção de confirmação deve estar marcada, o campo de senha atual deve estar preenchido com a senha 'senhasecreta'.
      • Em caso de falha: Use fail(400, { ... }) com o erro "Você deve confirmar a desativação".
      • Em caso de sucesso: Use redirect(303, '/07/login?status=conta_desativada') para deslogar o usuário e enviá-lo para a página de login.

    Crie o arquivo src/routes/07/login/+page.svelte para receber o redirecionamento após a desativação de conta, e que exibe uma mensagem caso contenha o parâmetro de consulta status com o valor conta_desativada.

  2. Você está criando um sistema de gerenciamento de conteúdo (CMS). Na página de edição de um artigo, o autor precisa de três opções: publicar o artigo, salvá-lo como rascunho ou excluí-lo.

    Crie o arquivo src/routes/07/blog/[id]/editar/+page.svelte com um formulário com os seguintes campos e botões:

    • Um campo para o Título do artigo
    • Uma área de texto para o Conteúdo do artigo
    • Um campo oculto para o ID do artigo

    Crie o arquivo src/routes/07/blog/[id]/editar/+page.server.js com as seguintes ações nomeadas:

    1. Ação publicar:

      • Validação: ID, Título e Conteúdo não podem estar vazios.
      • Falha: fail(400, ...) com a mensagem de erro e os dados preenchidos.
      • Sucesso: redirect(303, '/07/blog/[id]?status=artigo_publicado'), onde [id] é o id do artigo.

    2. Ação salvarRascunho:

      • Validação: Apenas o Título é obrigatório. O conteúdo pode estar vazio.
      • Falha: fail(400, ...) se o título estiver vazio.
      • Sucesso: Não redirecione. Retorne um objeto de sucesso { success: true, message: 'Rascunho salvo com sucesso!' } para que a página exiba a mensagem sem recarregar.

    3. Ação excluir:

      • Validação: Garanta que o ID do artigo foi enviado.
      • Falha: fail(400, ...) se o ID estiver ausente.
      • Sucesso: redirect(303, '/07/blog?status=artigo_excluido').

    Crie o arquivo src/routes/07/blog/[id]/+page.svelte para receber o redirecionamento após a publicação do artigo, e que tenha um conteúdo fixo fictício. Caso haja o parâmetro de consulta status com o valor artigo_publicado, exiba uma mensagem correspondente.

    Crie o arquivo src/routes/07/blog/+page.svelte para receber o redirecionamento após a exclusão do artigo, e que tenha um conteúdo fixo fictício. Caso haja o parâmetro de consulta status com o valor artigo_excluido, exiba uma mensagem correspondente.

  3. Recentemente, três novas irmandades abriram inscrições em seu reino de fantasia. Para se juntar a cada uma dessas irmandades, existem pré-requisitos específicos:
    • Ordem dos Guerreiros: Mínimo de 18 anos; Força física ≥ 7 (escala de 1 a 10).
    • Círculo dos Magos: Mínimo de 16 anos; Inteligência ≥ 8; Possui "conhecimento de magia" (checkbox).
    • Guilda dos Artesãos: Mínimo de 15 anos; Habilidade manual ("destreza") ≥ 6; Possui "ferramentas de artesão" (checkbox).

    Crie uma página de inscrição que permita ao usuário tentar se registrar em qualquer uma das três irmandades:

    • Cada botão do formulário deve acionar uma ação nomeada diferente, que validará os requisitos da irmandade correspondente.
    • Se o usuário atender aos requisitos, a ação deve redirecionar para a página inicial da respectiva irmandade (que você também deverá implementar).
    • Se faltar qualquer requisito, a ação deve retornar um erro via fail(), preservando os campos preenchidos e exibindo a mensagem de erro apropriada.

    Estrutura de Rotas:

    • src/routes/07/irmandades/+page.svelte: Página de inscrição.
    • src/routes/07/irmandades/guerreiros/+page.svelte: Página inicial dos guerreiros.
    • src/routes/07/irmandades/magos/+page.svelte: Página inicial dos magos.
    • src/routes/07/irmandades/artesaos/+page.svelte: Página inicial dos artesãos.
  4. Você trabalha em um banco digital que oferece três tipos de empréstimo, cada um com pré-requisitos diferentes:
    • Empréstimo Pessoal: Renda mensal ≥ R$ 2.000; Score de crédito ≥ 600.
    • Empréstimo Imobiliário: Renda mensal ≥ R$ 5.000; Score de crédito ≥ 700; Possui imóvel para garantia (checkbox).
    • Empréstimo Automotivo: Renda mensal ≥ R$ 3.000; Score de crédito ≥ 650; Valor do veículo ≥ R$ 15.000.

    Crie uma página de solicitação de empréstimo que permita ao usuário escolher qual modalidade deseja:

    • Cada botão do formulário deve acionar uma ação nomeada diferente, que validará os requisitos da modalidade correspondente.
    • Se o usuário atender aos requisitos, a ação deve redirecionar para a página de confirmação daquele tipo de empréstimo (que você também deverá implementar).
    • Se faltar qualquer requisito, a ação deve retornar um erro via fail(), preservando os campos preenchidos e exibindo a mensagem de erro apropriada.

    Estrutura de Rotas:

    • src/routes/07/emprestimo/+page.svelte: Página de solicitação de empréstimo.
    • src/routes/07/emprestimo/pessoal/+page.svelte: Página de confirmação do empréstimo pessoal.
    • src/routes/07/emprestimo/imobiliario/+page.svelte: Página de confirmação do empréstimo imobiliário.
    • src/routes/07/emprestimo/automotivo/+page.svelte: Página de confirmação do empréstimo automotivo.
