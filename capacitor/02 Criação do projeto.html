Criando um projeto SvelteKit com Capacitor, Drizzle ORM e SQLite

Criando

Primeiramente abra o terminal na pasta onde deseja criar o projeto.

Execute os seguintes comandos:

pnpm dlx sv create meu-projeto-capacitor
cd meu-projeto-capacitor/
pnpm add -D @capacitor/cli drizzle-kit
pnpm add @capacitor/core @capacitor/android @capacitor/filesystem drizzle-orm sql.js bootstrap bootstrap-icons
npx cap init meu-projeto-capacitor io.github.pedrosiqueira.meuProjetoCapacitor
npx cap add android
code -r .

Configurando

  1. modifique o valor da propriedade "webDir" do arquivo capacitor.config.json para "build":

    {
  "appId": "io.github.pedrosiqueira.svelteCapacitorDrizzleSqlite",
  "appName": "svelte-capacitor-drizzle-sqlite",
  "webDir": "build"
}

  2. Crie ou edite o arquivo src/routes/+layout.js, adicionando as seguintes linhas:

    export const prerender = true;
export const ssr = false;

Explicação da configuração

Local do build

Por padrão, o CapacitorJS espera que os arquivos do projeto construído estejam na pasta dist. No entanto, projetos criados com Svelte (via vite build) geram os arquivos finais na pasta build.

Para que o Capacitor encontre corretamente os arquivos do app, modificamos o valor da propriedade "webDir" do arquivo capacitor.config.json para "build".

Opções de página

uma restrição de html+css+js com capacitor, é que não temos um servidor back-end rodando. O Capacitor executa o app em um WebView embarcado no Android (ou iOS), carregando o conteúdo do app a partir de arquivos estáticos (HTML, CSS, JS). tudo tem q ser feito no lado do cliente. portanto, nosso app é um "site" [SPA](https://svelte.dev/docs/kit/single-page-apps): [explique o que é SPA].

Opção prerender

No SvelteKit, para tornar o app um SPA, podemos configurá-lo com o adaptador estático, que fizemos no momento da criação do projeto, e [adicionar a linha](https://svelte.dev/docs/kit/page-options#prerender) `export const prerender = true` no arquivo `src/routes/+layout.js`. Isso fará com que todas as páginas sejam geradas de antemão no momento do build (tempo de compilação), e não sob demanda no momento da execução (tempo de execução) — ou seja, gerar arquivos HTML e JS estáticos prontos para serem servidos, sem depender de um servidor backend em tempo real.

Not all pages can be prerendered. [The basic rule is this](https://svelte.dev/docs/kit/glossary#Prerendering): for content to be prerenderable, any two users hitting it directly must get the same content from the server, and the page must not contain form actions. Para nós, isso não vai ser problema, pois form actions exigem um servidor back-end, que não utilizaremos.

Opção ssr

Mesmo com a opção `export const prerender = true`, antes dos componentes `.svelte` serem carregados no cliente, o script do componente é executado no servidor, num processo chamado de [hidratação](https://svelte.dev/docs/kit/page-options). como vamos carregar scripts bootstrap em algumas páginas, que acessam a variável global `document`, que está disponível apenas no lado do cliente, precisamos [adicionar a linha](https://svelte.dev/docs/kit/page-options#ssr) `export const ssr = false;` no arquivo `src/routes/+layout.js` para que o javascript não seja executado no lado do servidor.

Conclusão

Com essas duas diretivas:

  • **A página será gerada como HTML estático no momento do build**, mas **toda a lógica de carregamento e renderização ocorrerá no cliente.**
  • Isso é típico para **SPAs (Single Page Applications)** onde não há necessidade de SSR.

Explicação dos comandos

pnpm dlx sv create meu-projeto-capacitor
  • Usa pnpm dlx para rodar o comando sem instalar globalmente.
  • sv create meu-projeto-capacitor → Cria um novo projeto Svelte chamado meu-projeto-capacitor.
cd meu-projeto-capacitor/
  • Entra no diretório do projeto recém-criado.
pnpm add -D @capacitor/cli drizzle-kit
  • Instala como dependência de desenvolvimento (-D):
    • @capacitor/cli → CLI do Capacitor (para gerar builds e rodar comandos).
    • drizzle-kit → Ferramenta para geração e migração de banco de dados Drizzle.

(Essas dependências são usadas apenas durante o desenvolvimento, e não são incluídas na versão final do aplicativo.)

pnpm add @capacitor/core @capacitor/android @capacitor/filesystem drizzle-orm sql.js bootstrap bootstrap-icons
  • Instala como dependência do projeto:
    • @capacitor/core → Core do Capacitor, necessário para qualquer app Capacitor.
    • @capacitor/android → Plugin para gerar um app Android.
    • @capacitor/filesystem → API do Capacitor para manipular arquivos (útil para SQLite).
    • drizzle-orm → Biblioteca ORM para manipulação de banco de dados.
    • sql.js → Biblioteca que permite rodar SQLite na memória via WebAssembly.
    • bootstrap → Framework CSS para estilização.
    • bootstrap-icons → Ícones do Bootstrap.

(Essas dependências no código em tempo de execução (produção), e são incluídas na versão final do aplicativo.)

(altere para o teu nome de usuário do GitHub...)

npx cap init meu-projeto-capacitor io.github.pedrosiqueira.meuProjetoCapacitor
  • Inicializa o Capacitor no projeto.
  • Exigências para desenvolvimento de aplicativos Android:
    • Define o nome do app (meu-projeto-capacitor).
    • Define o ID do pacote (io.github.pedrosiqueira.meuProjetoCapacitor), que será usado na Play Store/App Store.
npx cap add android
  • Adiciona a plataforma Android ao projeto Capacitor.
  • Cria a pasta android/, onde o código nativo será gerado.
code -r .
  • Abre o projeto no VS Code (. significa abrir a pasta atual).
  • -r (reutilizar) → Se já houver uma instância aberta do VS Code, abre o projeto nela.

Resumo do que acontece

  1. Cria um projeto Svelte.
  2. Instala o Capacitor e Drizzle ORM.
  3. Adiciona SQLite via sql.js.
  4. Prepara o Capacitor para Android.
  5. Abre o projeto no VS Code.

Agora o projeto está pronto para desenvolvimento com Svelte + Capacitor + Drizzle ORM + SQLite.
