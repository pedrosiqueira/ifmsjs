Criando um projeto SvelteKit com Capacitor, Drizzle ORM e SQLite

Criando

Primeiramente abra o terminal na pasta onde deseja criar o projeto.

Execute os seguintes comandos:

pnpm dlx sv create meu-projeto-capacitor
cd meu-projeto-capacitor/
pnpm add -D @capacitor/cli drizzle-kit
pnpm add @capacitor/core @capacitor/android @capacitor/filesystem drizzle-orm sql.js bootstrap bootstrap-icons
npx cap init meu-projeto-capacitor io.github.pedrosiqueira.meuProjetoCapacitor
npx cap add android
code -r .

Configurando

  1. No arquivo capacitor.config.json, altere o valor da propriedade "webDir" para "build":

    {
  "appId": "io.github.pedrosiqueira.svelteCapacitorDrizzleSqlite",
  "appName": "svelte-capacitor-drizzle-sqlite",
  "webDir": "build"
}

  2. Crie ou edite o arquivo src/routes/+layout.js, adicionando as seguintes linhas:

    export const prerender = true;
export const ssr = false;

Explicação dos comandos

pnpm dlx sv create meu-projeto-capacitor
  • Usa pnpm dlx para rodar o comando sem instalar globalmente.
  • sv create meu-projeto-capacitor → Cria um novo projeto Svelte chamado meu-projeto-capacitor.

  • Entra no diretório do projeto recém-criado.
pnpm add -D @capacitor/cli drizzle-kit
  • Instala como dependência de desenvolvimento (-D):
    • @capacitor/cli → CLI do Capacitor (para gerar builds e rodar comandos).
    • drizzle-kit → Ferramenta para geração e migração de banco de dados Drizzle.

(Essas dependências são usadas apenas durante o desenvolvimento, e não são incluídas na versão final do aplicativo.)

pnpm add @capacitor/core @capacitor/android @capacitor/filesystem drizzle-orm sql.js bootstrap bootstrap-icons
  • Instala como dependência do projeto:
    • @capacitor/core → Core do Capacitor, necessário para qualquer app Capacitor.
    • @capacitor/android → Plugin para gerar um app Android.
    • @capacitor/filesystem → API do Capacitor para manipular arquivos (útil para SQLite).
    • drizzle-orm → Biblioteca ORM para manipulação de banco de dados.
    • sql.js → Biblioteca que permite rodar SQLite na memória via WebAssembly.
    • bootstrap → Framework CSS para estilização.
    • bootstrap-icons → Ícones do Bootstrap.

(Essas dependências no código em tempo de execução (produção), e são incluídas na versão final do aplicativo.)

Resumo do que acontece

  1. Cria um projeto Svelte.
  2. Instala o Capacitor e Drizzle ORM.
  3. Adiciona SQLite via sql.js.
  4. Prepara o Capacitor para Android.
  5. Abre o projeto no VS Code.

Agora o projeto está pronto para desenvolvimento com Svelte + Capacitor + Drizzle ORM + SQLite!

Explicação da Configuração

1. Local do Build

Por padrão, o CapacitorJS espera que os arquivos do projeto construído estejam na pasta dist. No entanto, projetos criados com Svelte (via vite build) geram os arquivos finais na pasta build.

Para que o Capacitor localize corretamente os arquivos do app, é necessário modificar a propriedade "webDir" no arquivo capacitor.config.json para "build".

2. Opções de Página

Ao usar Capacitor, estamos trabalhando com HTML, CSS e JavaScript executados dentro de um WebView — sem um servidor back-end local. O app é carregado a partir de arquivos estáticos, o que significa que toda a lógica precisa ser executada no lado do cliente. Isso caracteriza uma SPA (Single Page Application) — ou seja, um aplicativo em que todo o conteúdo é carregado dinamicamente sem recarregamento de página.

Opção prerender

No SvelteKit, para tornar o app um SPA, usamos o adaptador estático (configurado na criação do projeto) e adicionamos export const prerender = true; no arquivo src/routes/+layout.js.

Isso garante que todas as páginas sejam pré-renderizadas de antemão durante o build (tempo de compilação), e não sob demanda durante a execução (tempo de execução) — gerando arquivos HTML e JS estáticos prontos para serem usados, sem necessidade de um servidor back-end em tempo real.

Nem todas as páginas podem ser pré-renderizadas. A regra básica é: o conteúdo deve ser o mesmo para qualquer usuário que acesse diretamente a página, e ela não deve conter form actions. No nosso caso, isso não será um problema, pois não escreveremos código que dependa de back-end.

Opção ssr

Mesmo com a opção prerender, o SvelteKit ainda executaria o JavaScript dos componentes no servidor antes de enviá-lo ao cliente — processo conhecido como hidratação.

No entanto, como utilizaremos o Bootstrap, que acessa objetos do DOM (como document ou window), precisamos impedir essa execução no servidor. Para isso, adicionamos export const ssr = false; no arquivo src/routes/+layout.js, garantindo que tudo seja executado exclusivamente no cliente.

Resumo do que acontece

Com essas duas diretivas:

  • A página será gerada como HTML estático no momento do build,
  • Toda a lógica e renderização será feita no lado do cliente.

Essa abordagem é ideal para SPAs (Single Page Applications) sem necessidade de SSR (Server-Side Rendering).
